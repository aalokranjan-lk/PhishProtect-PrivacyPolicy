1. Overview

PhishProtect is a Chrome Extension designed to protect users from phishing threats in Gmail. It detects suspicious links using a combination of trusted domains, blacklist rules, punycode decoding, homograph detection, and typo-squatting analysis.

2. Information We Collect

PhishProtect does not collect any personally identifiable information (PII) or track user activity. The extension uses local browser storage to cache trusted domain lists and user-defined whitelists.

When a suspicious domain is hovered over, the extension may send the following non-sensitive metadata to our secure backend:

  • Hovered domain name
  • Timestamp
  • Email sender and recipient (extracted from the Gmail UI)

This is used solely for internal phishing analysis. No message content or sensitive data is captured.

3. How We Use Data

Any data processed is strictly used for:

  • Classifying suspicious URLs
  • Alerting internal security teams (optional)
  • Improving phishing detection heuristics

We never sell, rent, or share user data. No advertising or analytics services are used.

4. Chrome Permissions Justification

PhishProtect uses the following permissions:

  • storage: To store trusted domains and user whitelist settings
  • https://storage.googleapis.com/*: To fetch static domain classification TXT files
  • https://mail.google.com/*: To monitor links in Gmail
  • https://hooks.slack.com/*: (Optional) To send alerts to internal Slack channels

No remote JavaScript is executed. All fetched resources are parsed as text.

5. Data Security

All data sent to backend systems is transmitted over HTTPS. We use secure authentication and adhere to industry best practices to protect any metadata exchanged.

6. User Control

Users can manage their whitelist settings at any time via the extension's options page. Uninstalling the extension deletes all stored data. There is no telemetry, tracking, or analytics.
